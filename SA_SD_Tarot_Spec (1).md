# 塔羅抽牌 MVP（單檔 Index.html）— SA + SD 規格文件

- 文件類型：SA（系統分析）+ SD（系統設計）
- 目標版本：MVP
- 交付物：單檔 `index.html`（含基本 UI、隨機抽牌、正逆位、簡短敘述、紀錄列表）
- 執行環境：任意現代瀏覽器（Chrome / Edge / Safari / Firefox），可離線開啟

---

## 1. 背景與目的（SA）

塔羅牌常用於自我探索、情境梳理與決策輔助。本系統以「引導式反思」為定位，提供簡易抽牌與牌義簡述，讓使用者以最低成本完成一次抽牌流程，並可回顧結果。

---

## 2. 系統範圍（Scope）（SA）

### 2.1 範圍內（In Scope）
1. 牌陣選擇：單張、三張（過去/現在/未來）
2. 逆位開關：可選是否啟用逆位，啟用時每張 50% 機率逆位
3. 問題輸入（可選）
4. 隨機抽牌：不重複抽取
5. 結果呈現：牌名 + 正/逆位 + 簡短敘述 + 位置（若有）
6. 紀錄：頁面內記憶體保存（本次開啟期間），可清除

### 2.2 範圍外（Out of Scope / 後續擴充）
- 登入、跨裝置同步、後端 API
- 完整 78 張牌庫（MVP 可先用示範牌庫）
- 牌陣編輯器（自定義位置）
- AI 生成解讀、付費訂閱、諮詢師媒合

---

## 3. 角色與使用情境（SA）

### 3.1 角色（Actors）
- 使用者（User）：在單頁 UI 操作抽牌、查看結果、回顧紀錄

### 3.2 主要情境（Use Cases）
- UC-01：使用者選擇牌陣與逆位設定後抽牌，獲得當次解讀
- UC-02：使用者輸入問題後抽牌，並在結果中看到該問題
- UC-03：使用者在「紀錄」區回顧剛剛的占卜結果
- UC-04：使用者清除紀錄，回到初始狀態

---

## 4. 功能需求（Functional Requirements）（SA）

### FR-01 牌陣選擇
- 系統應提供至少 2 種牌陣：
  - 單張（指引）
  - 三張（過去/現在/未來）
- 使用者變更牌陣不會自動抽牌，需按「抽牌」才執行。

**驗收條件**
- 選擇「單張」時抽出 1 張；選擇「三張」時抽出 3 張，並顯示對應位置。

---

### FR-02 逆位開關
- 使用者可切換是否啟用逆位：
  - 若啟用：每張牌以 50% 機率逆位（可用常數調整）
  - 若不啟用：全部為正位

**驗收條件**
- 關閉逆位後抽牌結果不得出現「逆位」標記。

---

### FR-03 問題輸入
- 系統應提供文字輸入欄位供使用者輸入問題。
- 問題可為空白。

**驗收條件**
- 未輸入問題時，結果與紀錄顯示「（未輸入問題）」。

---

### FR-04 隨機抽牌（不重複）
- 系統抽牌須滿足：
  - 從牌庫中隨機抽取 N 張（N 依牌陣定義）
  - 同一次抽牌內不得重複同一張牌

**驗收條件**
- 三張牌陣抽出的 3 張 `card.id` 不得相同。

---

### FR-05 結果呈現
- 系統應顯示每張牌：
  - 位置標籤（單張可顯示「指引」）
  - 牌名
  - 正/逆位
  - 簡短敘述（正位取 upright；逆位取 reversed）

**驗收條件**
- 若逆位，敘述必須使用 reversed 字段而非 upright。

---

### FR-06 紀錄（History）
- 系統應在每次抽牌完成後新增一筆紀錄至紀錄列表。
- 紀錄內容需包含：
  - 抽牌時間
  - 牌陣
  - 問題
  - 各位置的牌名、正逆位、簡短敘述
- 系統提供「清除紀錄」按鈕可清空紀錄。

**驗收條件**
- 清除後，紀錄列表顯示「尚無紀錄」。

---

## 5. 非功能需求（Non-functional Requirements）（SA）

1. **可攜性**：單檔 HTML 可直接以瀏覽器開啟運行
2. **離線可用**：不依賴外部 CDN/後端
3. **相容性**：支援主流現代瀏覽器
4. **易維護**：牌庫資料可直接擴充（不需重寫抽牌/渲染邏輯）
5. **UX**：基本 RWD，手機可閱讀

## 6. 資料結構與領域模型（SD）

### 6.1 TarotCard（牌）
- `id: string`
- `name: string`
- `upright: string`（正位簡述）
- `reversed: string`（逆位簡述）
- `keywords?: string[]`（可選）

### 6.2 Spread（牌陣）
- `id: "one" | "three"`
- `name: string`
- `positions: Array<{ key: string; label: string }>`

### 6.3 Reading（一次抽牌結果）
- `id: string`
- `createdAt: string (ISO)`
- `spreadId: string`
- `question: string`
- `cards: Array<{
    positionKey: string;
    positionLabel: string;
    card: TarotCard;
    isReversed: boolean;
  }>`
- `notes?`（MVP 不做，可預留）

## 7. 系統架構（SD）

### 7.1 架構概述
- **UI 層**：HTML + CSS（同檔）
- **邏輯層**：JavaScript（同檔）
- **資料層**：牌庫常數 `TAROT_CARDS` + 牌陣定義 `SPREADS` + 記憶體狀態 `state.history`

> MVP 不使用後端、不做登入、不做本地持久化（localStorage）。若要持久化，新增 `loadHistory()/saveHistory()` 即可擴充。

## 8. UI 規格（SD）

### 8.1 控制區（左側）
- 牌陣下拉選單（單張 / 三張）
- 問題輸入框（可空）
- 逆位 checkbox（預設勾選）
- 按鈕：
  - 抽牌（主動作）
  - 重置輸入（重置牌陣/問題/逆位設定）
  - 清除紀錄（清空 history）

### 8.2 結果區（右側）
- 顯示 meta：時間、牌陣、問題
- 顯示卡片結果（依牌陣顯示 1 或 3 張）
- 顯示紀錄列表（最新在前）

---

## 9. 核心演算法（SD）

### 9.1 抽牌（不重複）
1. 輸入：`count = positions.length`
2. 使用 Set 紀錄已抽過 id
3. while result.length < count：
   - idx = randomInt(TAROT_CARDS.length)
   - 若 card.id 未出現：加入 result

### 9.2 逆位判定
- 若 `useReversed = true`：`isReversed = Math.random() < 0.5`
- 否則 `isReversed = false`

---

## 10. 錯誤與例外處理（SD）

- 若 `count > TAROT_CARDS.length`：
  - 丟出錯誤或顯示提示：「牌庫不足，無法抽取這麼多張不重複的牌」
- 若瀏覽器不支援 `crypto.randomUUID()`：
  - 使用 fallback：`"reading-" + new Date().toISOString()`

---

## 11. 驗收測試（Acceptance Tests）（SA+SD）

1. **單張抽牌**
   - 選單=單張，按抽牌 → 顯示 1 張結果，位置為「指引」
2. **三張抽牌**
   - 選單=三張，按抽牌 → 顯示 3 張結果，位置依序：過去/現在/未來
3. **不重複**
   - 三張中不得出現相同 `card.id`
4. **逆位關閉**
   - 逆位 checkbox 取消勾選 → 所有牌都顯示正位且使用 upright 敘述
5. **紀錄新增**
   - 每抽一次，紀錄列表新增一筆（最新在最上方）
6. **清除紀錄**
   - 按清除紀錄 → 紀錄列表顯示「尚無紀錄」，結果區清空或顯示尚未抽牌
